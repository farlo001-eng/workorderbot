ASSUMPTIONS & NOTES
--------------------
1. Excel date parsing: parse_date() handles three input types:
   - int/float  → treated as Excel serial date (epoch: Dec 30, 1899)
   - datetime   → formatted directly with strftime
   - string     → stripped and returned as-is
   The offset used is +693594-1 (standard xlrd/openpyxl epoch correction).

2. Employee column values assumed to be short keys ("asmith", "dhall", "morton")
   matching the tech_key field in /api/properties. If Yardi exports full names
   instead (e.g. "Antonio Smith"), the /api/workorders?employee= filter will need
   to match on full name, not tech_key.

3. "Technician Notes" column from Yardi is NOT mapped in sync_workorders.py —
   the sync endpoint intentionally preserves whatever tech_notes already exist
   in the DB (field-entered by the tech via the PWA). The Yardi export column
   was excluded by design.

4. The "Amount" and "Call Date Parsed" columns from the Yardi export are present
   in the header list but not mapped to any DB field — they are silently ignored
   during sync.

5. harvest-logo.jpg in static\ is a 1x1 JPEG placeholder. Replace with the real
   Harvest Property Management logo before deploying.

6. No virtual environment was created per instructions.
